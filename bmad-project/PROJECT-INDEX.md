## 🎯 Core Project Information

### Project Details
- **Name**: Beacon Buddy
- **Type**: Mobile App + SaaS B2C Hybrid
- **Domain**: Social Networking (Activity Discovery)
- **Complexity**: Medium-High
- **Context**: Brownfield (existing codebase)

### Technology Stack
- **Frontend**: React Native + Expo Router + TypeScript
- **Backend**: Ruby on Rails + PostgreSQL + Redis
- **Infrastructure**: Docker + Nginx + Cloud Hosting
- **Design**: Component-based design system

### Target Market
- **Initial**: Nagpur, India (300-500 users)
- **Expansion**: Multi-city rollout
- **Users**: Activity seekers, creators, corporate clients
- **Focus**: Introvert-friendly social discovery

## 🔧 Quick Reference

### Design System Components
- **Buttons**: PrimaryButton, SecondaryButton, GhostButton
- **Cards**: Card, BeaconCard, ActivityCard
- **Forms**: Input, SearchBar, TextArea, Select
- **Display**: Avatar, Badge, Chip, Progress
- **Layout**: Container, Section, Modal
- **Navigation**: Header, TabBar, Breadcrumb

### API Endpoints
- **Authentication**: `/api/v1/auth/google`
- **Users**: `/api/v1/users`, `/api/v1/users/:id`
- **Beacons**: `/api/v1/beacons`, `/api/v1/beacons/:id`
- **Participants**: `/api/v1/participants`
- **Messages**: `/api/v1/messages`

### Database Models
- **User**: Authentication, profiles, reliability
- **Beacon**: Activities, locations, participants
- **Participant**: Join requests, attendance
- **Message**: Ephemeral chat
- **Reliability**: Score history
